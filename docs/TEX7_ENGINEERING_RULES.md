# TEX7 Engineering Rules — Gallery 015

## UI & Design Principles
- No AI-looking design.
- No generic icons.
- No emoji in UI.
- No fake futuristic gradients.
- No unnecessary animations.
- No inline styles.
- Public UI must feel like an art institution, not a startup template.
- Admin UI must feel like a professional internal operating system.

## Development Rules
- One patch = one purpose.
- Maximum 5 files modified per patch unless explicitly approved.
- No refactor unless requested.
- Stop after requested scope.
- Report changed files only.
- Do not modify data files unless part of a data-specific patch.
- Keep components under 100 lines where possible.
